# Protection

`protection.sh` - интерактивный bash-скрипт для базовой настройки и администрирования Ubuntu/Debian серверов. Скрипт помогает быстро настроить SSH, пользователей, firewall, Fail2ban, Docker, Ollama и сохранить важные параметры в `info.txt`.

> Внимание: скрипт меняет системные настройки сервера. Запускайте его только на сервере, к которому у вас есть резервный способ доступа.

## Возможности

- изменение пароля `root`;
- создание пользователей и настройка SSH-ключей;
- изменение SSH-порта;
- включение/отключение входа `root` по SSH;
- включение/отключение ответа на ICMP ping;
- установка и управление Fail2ban;
- настройка и управление UFW firewall;
- установка, обновление и обслуживание Docker;
- установка Ollama и модели `nomic-embed-text`;
- автоматическое создание глобальной команды `protection`.

## Требования

- Ubuntu или Debian;
- запуск от `root`;
- доступ к интернету для установки пакетов;
- `sudo` необязателен: если он установлен, скрипт использует его, иначе выполняет команды напрямую от `root`.

## Установка

Скачайте скрипт на сервер:

```bash
curl -fsSL -o protection.sh https://raw.githubusercontent.com/svu2009-prog/Protection/main/protection.sh
chmod +x protection.sh
```

Запустите:

```bash
./protection.sh
```

При запуске скрипт создаёт или обновляет глобальную команду:

```bash
/usr/local/bin/protection
```

Команда устанавливается копированием самого скрипта в `/usr/local/bin/protection`, без символической ссылки и без wrapper-файла. Если установленная версия совпадает с текущей, файл не перезаписывается. Если установленная версия старее, она заменяется новой копией.

После первого запуска можно использовать:

```bash
protection
```

Проверить версию:

```bash
protection --version
```

## Главное меню

Основное меню открывается при запуске `protection` или `./protection.sh` и содержит действия:

1. создать пользователя;
2. поменять пароль `root`;
3. поменять порт SSH;
4. сохранить информацию в `info.txt`;
5. отключить или включить ping;
6. показать справку;
7. настроить SSH-ключи;
8. настроить Fail2ban;
9. настроить Firewall UFW;
10. отключить или включить вход `root` по SSH;
11. открыть меню Docker;
12. установить Ollama и модели.

## Важные предупреждения

- Перед изменением SSH-порта убедитесь, что новый порт открыт в firewall и у вас есть активная сессия.
- Перед включением UFW проверьте, что текущий SSH-порт разрешён.
- Полное удаление Docker удаляет данные Docker из `/var/lib/docker`.
- Ошибочная настройка SSH-ключей или `PermitRootLogin` может привести к потере доступа.
- Файл `info.txt` может содержать чувствительные данные, включая пароли и порты. Не публикуйте его.

## Обновление

Скачайте свежую версию `protection.sh` и запустите её:

```bash
sudo curl -fsSL -o protection.sh https://raw.githubusercontent.com/svu2009-prog/Protection/main/protection.sh
sudo chmod +x protection.sh
sudo ./protection.sh
```

Если глобальная команда `/usr/local/bin/protection` старее скачанного скрипта, она будет заменена новой копией.
